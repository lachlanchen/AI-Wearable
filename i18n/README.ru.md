# **omi**

_AI-устройство для быстрых встреч и чатов: записывайте, анализируйте и действуйте — без использования рук._

Познакомьтесь с Omi — ведущим открытым AI-носителем, который записывает разговоры, делает сводки, выделяет задачи и выполняет действия за вас. Просто подключите Omi к мобильному устройству и получите автоматическую высококачественную транскрипцию встреч, чатов и голосовых заметок где бы вы ни находились.


## Требования

Используйте требования, относящиеся к вашему целевому компоненту.

| Инструмент | Используется для |
| --- | --- |
| Git | Управление исходным кодом |
| Node.js 18+ и npm | веб, personas, некоторые рабочие процессы плагинов |
| Python 3.10+ | backend, MCP, Python SDK |
| Flutter SDK | мобильное приложение |
| Xcode/CocoaPods | разработка iOS |
| Android Studio + Android SDK | разработка Android |
| `ffmpeg` + `opus` | backend/audio-процессы |
| Docker (опционально) | развертывание MCP/web |
| Google Cloud + Firebase | backend/workflows приложения |

### Допущения для настройки

- Некоторые команды зависят от внешних учетных данных (Firebase, OpenAI/Deepgram/API-провайдеры), которые обычно настраиваются для каждого компонента через его шаблон `.env`. Если в локальной документации отсутствуют значения, см. README соответствующего компонента.
- Если вам нужен только один интерфейс, устанавливайте и собирайте только этот компонент, чтобы ускорить итерации.

## Установка

Так как это монорепозиторий, устанавливайте по компонентам.

### Клонирование репозитория

```bash
git clone https://github.com/BasedHardware/omi.git
cd omi
```

### Приложение (Flutter)

```bash
cd app
bash setup.sh ios
# или
bash setup.sh android

flutter run --flavor dev
```

### Backend (FastAPI)

```bash
cd backend
cp .env.template .env
pip install -r requirements.txt
uvicorn main:app --reload --env-file .env
```

### Основной frontend (Next.js)

```bash
cd web/frontend
npm install
cp .env.template .env.local
npm run dev
```

### Personas web app (Next.js)

```bash
cd web/personas-open-source
npm install
npm run dev
```

### MCP-сервер

```bash
cd mcp
# См. mcp/README.md для настройки Claude Desktop через Docker
```

### SDK

```bash
cd sdks/python && pip install -e .
cd ../react-native && npm install
# Swift-пакет доступен из корневого Package.swift как `omi-lib`
```

## Использование

### Поток для конечного пользователя

1. Установите приложение Omi по ссылкам выше.
2. Подключите/используйте устройство Omi.
3. Создайте приложение через быстрый старт webhook.
4. Говорите и наблюдайте события транскрипции в реальном времени на вашем webhook endpoint.

### Локальный backend с приложением

1. Запустите backend локально на порту `8000`.
2. Откройте локальный backend через статический домен ngrok.
3. Установите `BASE_API_URL` в приложении на URL вашего ngrok.
4. Используйте Omi-приложение с локальным backend.

### Потоки SDK

- Python SDK: сканируйте BLE-устройства с помощью `omi-scan`, декодируйте Opus-пакеты и выполняйте транскрипцию через Deepgram.
- React Native SDK: подключайтесь, стримьте аудио-байты, проверяйте кодек/уровень батареи.
- Swift SDK (`omi-lib`): сканируйте/подключайтесь/делайте транскрипцию через API `OmiManager`.

## Конфигурация

Конфигурация распределена по компонентам. Используйте шаблоны и README соответствующего подпроекта как источник истины.

- `backend/.env.template` → ключи API backend и конфигурация инфраструктуры
- `web/frontend/.env.template` → переменные окружения веб-фронтенда
- Значения `app` → конфигурация среды выполнения мобильного приложения
- Переменные `mcp` → `OMI_API_KEY` и необязательный `OMI_API_BASE_URL`
- Специфичные для SDK переменные:
  - Python SDK использует `DEEPGRAM_API_KEY`
  - Personas использует переменные Firebase/OpenRouter/RapidAPI/Mixpanel

Если вы не уверены, какие переменные нужны, оставьте значения по умолчанию и сверяйтесь с конкретным README компонента перед удалением/изменением значений.

## Заметки по разработке

- CI/CD и автоматизация деплоя настроены в:
  - `.github/workflows/`
  - `codemagic.yaml`
- Скрипт `scripts/pre-commit` в корне форматирует подготовленные файлы для:
  - Dart в `app/`
  - Python в `backend/`
  - C/C++ в `omi/` и `omiGlass/`
- `Package.swift` в корне репозитория публикует Swift-пакет `omi-lib` из `sdks/swift`.
- Каталог `i18n/` существует и предназначен для мультиязычных вариантов README/контента.

## Устранение неполадок

- Проблема с моделью/сетевым доступом при локальной настройке backend: если вы сталкиваетесь с известной ошибкой SSL-подкачки, примените обходной путь `ssl`, описанный в `backend/README.md`, и повторите попытку.
- Приложение не может достучаться до локального backend: проверьте, что домен ngrok запущен и `BASE_API_URL` указывает ровно на этот публичный URL.
- Проблемы с обнаружением BLE: убедитесь, что разрешения Bluetooth выданы для вашей платформы (Terminal/Xcode/Android-приложение, где это применимо).
- Несовпадение путей в старой документации: в некоторых устаревших текстах могут встречаться старые пути (например, путь `personas-open-source` в корне репозитория или регистр `omi/OmiGlass`). В этом репозитории используйте текущую структуру папок, показанную выше.
